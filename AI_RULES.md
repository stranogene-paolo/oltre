# AI Rules (OLTRE)

- Non cambiare comportamento se non richiesto.
- Patch piccole: 1–2 file per volta.
- Niente nuove dipendenze/package senza richiesta.
- Niente refactor/rename massivi.
- Ogni patch deve includere: motivo, cosa cambia, come testare in Unity.
- Niente segreti/API key nel codice o nei prompt.
